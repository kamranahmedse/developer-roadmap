# When you find a bug, ask AI to write a breaking test and then fix

When a bug shows up, don't just ask the AI to fix it straight away. Instead, first ask it to write a test that reproduces the bug â€” a test that currently fails because of the broken behavior. Then ask it to fix the code so that the test passes. This approach is valuable for two reasons. First, it confirms that the AI actually understands what the bug is before it tries to fix it. Second, it leaves you with a permanent test that will alert you if that same bug ever comes back in the future. It turns every bug into a long-term improvement to your project's stability.
