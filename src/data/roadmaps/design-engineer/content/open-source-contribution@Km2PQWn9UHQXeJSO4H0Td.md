# Open Source Contribution

Contributing to open source builds skills, reputation, and connections. Many Design Engineers get hired through their open source work.

**Where to Start**:
- **Documentation**: Fix typos, improve examples, add guides
- **Bug fixes**: Small, well-defined issues
- **Design system contributions**: Components, tokens, patterns
- **Tools you use**: Improve what you work with daily

**Good First Contributions**:
- shadcn/ui: Add components, fix styling issues
- Radix UI: Documentation, examples
- Design system repos: Carbon, Primer, Polaris
- Animation libraries: Framer Motion examples

**Contribution Process**:
1. Find an issue or improvement
2. Fork the repository
3. Create a branch for your changes
4. Make changes with clear commits
5. Open a Pull Request with description
6. Respond to feedback constructively

**Building Reputation**:
- Quality over quantity
- Be helpful in discussions
- Maintain what you create
- Share your contributions publicly

**Your Own Projects**:
- Templates and starters
- Component libraries
- CLI tools
- Learning resources
